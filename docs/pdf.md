# PDF Renderer

[← Главная](index.md) · [fwui-native](fwui-native.md) · [performance](performance.md)

Нативный генератор PDF из Node tree. Без промежуточного HTML, без внешних subprocess. Обходит дерево напрямую и генерирует PDF через [libharu](https://github.com/libharu/libharu).

## Сборка

PDF рендерер включен по умолчанию (`FWUI_BUILD_PDF=ON`):

```bash
cmake -B build -DFWUI_BUILD_PDF=ON -DFWUI_BUILD_TESTS=ON
cmake --build build
```

Для отключения:

```bash
cmake -B build -DFWUI_BUILD_PDF=OFF
```

При включенном PDF определяется макрос `FWUI_HAS_PDF=1`.

## C++ API

### Базовое использование

```cpp
#include <fwui/fwui.hpp>
using namespace fwui;

auto doc = div({
    h1("Report Title"),
    p("Generated by FWUI PdfRenderer."),
});

// Рендер в файл
PdfRenderer::RenderToFile(doc, "report.pdf", {});

// Рендер в строку (binary PDF data)
std::string pdf_data = PdfRenderer::RenderToString(doc);
```

### Options

```cpp
PdfRenderer::Options opts;
opts.page_size = PageSize::A4();         // A4, Letter, A3, Custom(w, h)
opts.margins = {72, 72, 72, 72};         // top, right, bottom, left (points)
opts.fonts.default_size = 12.0f;         // базовый размер шрифта
opts.title  = "My Report";              // PDF metadata
opts.author = "FWUI";
opts.auto_page_numbers = true;
opts.page_number_format = "Page {page} of {total}";

PdfRenderer renderer(opts);
auto pdf = renderer.Render(doc);
```

### Размеры страниц

| Метод | Размер (pt) | Примерно |
|-------|-------------|----------|
| `PageSize::A4()` | 595 × 842 | 210 × 297 мм |
| `PageSize::Letter()` | 612 × 792 | 216 × 279 мм |
| `PageSize::A3()` | 842 × 1191 | 297 × 420 мм |
| `PageSize::Custom(w, h)` | произвольный | — |

### Колонтитулы

```cpp
opts.header = [](int page_num, int total) -> Element {
    return text("My Document — Page " + std::to_string(page_num));
};
opts.footer = [](int page_num, int total) -> Element {
    return text("Confidential");
};
```

Колонтитулы рендерятся вторым проходом после полной компоновки, поэтому `total` содержит финальное количество страниц.

### TrueType шрифты

Встроенные шрифты (Helvetica, Courier) поддерживают только Latin-1. Для кириллицы и других скриптов нужны TrueType:

```cpp
opts.fonts.ttf_regular     = "/path/to/font.ttf";
opts.fonts.ttf_bold        = "/path/to/font-bold.ttf";
opts.fonts.ttf_italic      = "/path/to/font-italic.ttf";
opts.fonts.ttf_bold_italic = "/path/to/font-bolditalic.ttf";
opts.fonts.ttf_mono        = "/path/to/mono.ttf";
```

## Поддерживаемые элементы

| Элемент | Реализация |
|---------|-----------|
| `h1`-`h6` | Размер 28-12pt, bold, с отступами |
| `p` | Word-wrap по ширине страницы |
| `strong`/`em` | Bold / italic инлайн |
| `code` (inline) | Courier, 90% размера |
| `pre` | Courier, серый фон, без wrap |
| `blockquote` | Indent + серая полоса слева + italic |
| `ul`/`ol` + `li` | Буллеты • / нумерация 1. 2. 3. |
| `table`/`thead`/`tbody`/`tr`/`th`/`td` | Пропорциональные колонки, рамки, серый заголовок |
| `img` | JPEG embedding (PNG при наличии libpng) |
| `hr` | Горизонтальная линия |
| `br` | Перенос строки |
| `div`/`section`/`article` | Контейнеры (рекурсивный обход) |

### Inline стили

Поддерживаются через `Node::SetStyle()`:

- `color` — цвет текста (`#RGB` или `#RRGGBB`)
- `font-size` — размер шрифта в pt
- `font-weight: bold` — жирный
- `font-style: italic` — курсив

## CLI: fwui-pdf

```bash
fwui-pdf [options]
  -i, --input <file.json>   JSON node tree (default: stdin)
  -o, --output <file.pdf>   Выходной PDF (default: output.pdf, - для stdout)
  --page-size <A4|Letter|A3> Размер страницы
  --title <title>           PDF metadata
  --author <author>         PDF metadata
  --margins <t,r,b,l>       Margins в points
  --font-size <pt>          Базовый размер шрифта
  --page-numbers            Включить нумерацию страниц
  --page-format <fmt>       Формат (default: "Page {page} of {total}")
```

Примеры:

```bash
# JSON → PDF
echo '{"tag":"h1","text":"Hello"}' | fwui-pdf -i - -o hello.pdf

# С метаданными и нумерацией
fwui-pdf -i doc.json -o report.pdf --title "Report" --page-numbers

# Letter, кастомные margins
fwui-pdf -i data.json -o us.pdf --page-size Letter --margins 54,54,54,54
```

## Ruby

```ruby
require 'fwui'
extend FWUI::DSL

doc = div([
  h1("Report"),
  p("Generated by FWUI."),
  table([
    tr([th("Name"), th("Value")]),
    tr([td("Alpha"), td("100")]),
  ])
])

# В файл
doc.to_pdf("report.pdf", title: "Report", page_numbers: true)

# В строку
pdf_data = doc.to_pdf(title: "Report", author: "FWUI")
File.binwrite("output.pdf", pdf_data)
```

`to_pdf` вызывает `fwui-pdf` CLI через subprocess. Убедитесь что `fwui-pdf` в `PATH` или укажите полный путь.

## Архитектура

```
Node tree
  └── PdfRenderer::Render()
        ├── FontManager (Helvetica/Courier families, TrueType)
        ├── LayoutEngine
        │   ├── LayoutNode() — dispatch по тегу
        │   ├── DrawWrappedSegments() — word-wrap с inline formatting
        │   ├── LayoutTable() — column measurement + proportional widths
        │   ├── LayoutImage() — JPEG/PNG embedding
        │   └── NewPage() / EnsureSpace() — page break management
        └── Finalize() — second pass: headers, footers, page numbers
```

Координатная система: libharu использует bottom-left origin. LayoutEngine работает top-down (cursor_y уменьшается), конвертируя при отрисовке.
